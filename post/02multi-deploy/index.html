问：Ansible 如何同时部署多台机器？

答：

Ansible 是一个自动化运维工具，可以用来管理多台机器的配置和部署。使用 Ansible 可以方便地实现同时部署多台机器的操作，以下是一个简单的示例：

  1. 在 Ansible 控制机上安装 Ansible，具体安装方法可以参考官方文档。

  2. 在 Ansible 控制机上创建一个 inventory 文件，用于定义要管理的机器列表，例如：

[web]
web1 ansible_host=192.168.1.10
web2 ansible_host=192.168.1.11
web3 ansible_host=192.168.1.12

上面的示例定义了一个名为 web 的主机组，包含三个机器 web1、web2 和 web3，它们的 IP 地址分别为 192.168.1.10、192.168.1.11 和 192.168.1.12。
  1. 在 Ansible 控制机上编写一个 Playbook 文件，用于定义要执行的任务，例如
- name: 部署监控脚本
  hosts: web
  tasks:
    - name: 拷贝监控脚本到远程机器
      copy:
        src: /path/to/monitor.sh
        dest: /home/user/monitor.sh
        mode: "0755"
    - name: 运行监控脚本
      shell: /home/user/monitor.sh

上面的示例定义了一个名为 “部署监控脚本” 的任务，它将会在 web 主机组中的所有机器上执行。任务的第一个步骤是将本地的 monitor.sh 脚本拷贝到远程机器的 /home/user 目录下，并赋予可执行权限。任务的第二个步骤是运行远程机器上的 monitor.sh 脚本。

  1. 在 Ansible 控制机上运行 ansible-playbook 命令，指定要执行的 Playbook 文件和 inventory 文件，例如：
ansible-playbook -i inventory playbook.yml

上面的命令将会执行名为 playbook.yml 的 Playbook 文件，同时使用名为 inventory 的 inventory 文件来指定要管理的机器列表。

通过上述步骤，就可以方便地使用 Ansible 同时部署多台机器。另外，Ansible 还提供了一些高级的功能，例如使用变量来管理不同机器的配置、使用模板来生成配置文件、使用角色来组织任务和变量等。这些功能可以帮助开发者更加方便和灵活地管理多台机器的配置和部署。
